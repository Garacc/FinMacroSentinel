# FinMacroSentinel 自动化财经时报与投资决策辅助系统 - 需求文档 (PRD)

## 1. 总体定位与架构设计

- **系统目标**：构建一个"信息采集 → 逻辑解析 → 投资推演 → 结构化推送 → 持久化沉淀"的全自动闭环系统
- **设计原则**：事实与观点分离；宏观驱动微观；强调风险提示与边界条件；本地数据与远程展示完全解耦；严格溯源
- **核心基调**：摒弃散户化的情绪噪音，以资深宏观对冲基金经理的视角，输出结构化、重风控、强逻辑的投资洞察

## 2. 核心功能模块与技术需求

### 2.1 任务调度与触发模块（Time & Trigger）

- **执行逻辑**：基于北京时间（UTC+8），设置严格的Cron定时任务
- **触发时间节点**：
  - 09:00 (盘前预演)：定调当日宏观环境、隔夜外盘映射及早盘风险敞口
  - 12:30 (午间复盘)：评估早盘资金流向、异动行业及情绪指标
  - 21:00 (夜盘/外盘前瞻)：聚焦欧美宏观数据发布、地缘政治动向及大宗商品/加密资产夜间波动预期

### 2.2 信息采集与处理模块（Content & Logic）

Agent需从指定的高质量数据源（如彭博、路透、华尔街见闻、主流研报库等）拉取信息，并按三大主题进行严格分类。

**🔴 投资建议生成规范（系统刚性约束）**：

1. **事实溯源**：每一条核心事实必须附带原始新闻的真实超链接（URL），严禁捏造虚假链接
2. **宏观/中观传导**：该事件如何影响产业链或宏观资金面？
3. **资产表现映射**：利好/利空哪些大类资产或行业ETF？（严禁推荐单一股票代码）
4. **时效与级别**：明确属于"短期情绪冲击"、"中期基本面拐点"还是"长期战略配置"因素
5. **风险控制**：必须列出该逻辑可能失效的"边界条件"及对冲方案

### 2.3 数据持久化存储模块（Storage）

- **存储机制**：推送飞书前，Agent必须在本地服务器指定路径下生成Markdown文件
- **命名规范**：YYYYMMDDHH.md（例：2026年2月20日上午9点推送的内容，文件名为 2026022009.md）
- **内容格式**：Markdown结构需包含各级标题与无序/有序列表，确保本地归档具备高可读性，并为后续RAG（检索增强生成）提供高信噪比语料

### 2.4 飞书渠道推送模块（Delivery & API Struct）

- **推送目标**：专用的飞书"财经时报"话题群（Topic Group）
- **交互形式**：卡片（Message Card）进行推送。每次推送生成一个完整使用飞书消息的话题卡片
- **数据结构约束**：
  - Agent必须调用飞书 POST /open-apis/im/v1/messages 接口（或旧版 webhook），msg_type 必须指定为 interactive
  - 严禁发送本地绝对路径或相对路径，卡片内容必须是经大模型处理后的纯Markdown文本

**【飞书话题群卡片 JSON Payload 标准示例】**

Agent开发时需严格遵循以下卡片结构生成请求体：

```json
{
  "config": {
    "wide_screen_mode": true
  },
  "header": {
    "title": {
      "tag": "plain_text",
      "content": "📈 FinMacroSentinel 财经时报"
    },
    "template": "blue"
  },
  "elements": [
    {
      "tag": "div",
      "text": {
        "tag": "markdown",
        "content": "【Markdown内容】"
      }
    }
  ]
}
```

## 3. 核心逻辑推理指令（Core Prompt Engineering）

开发Agent在调用大模型生成内容时，必须将以下内容设为 System Prompt。

**【System Prompt】**

```
Role:
你是一位拥有20年经验的华尔街宏观对冲基金经理兼首席风控官。你精通宏观经济学、周期理论与资金博弈，拥有严密的概率思维。

Task:
阅读输入的杂乱新闻数据，过滤短期情绪噪音，精准提取"宏观金融"、"行业中观"与"地缘政治"三大主线信息，推演其对大类资产的实质性影响，并务必提供真实的超链接溯源。

Strict Rules:
1. 拒绝预测，只做推演：杜绝使用"必将大涨"等散户词汇，使用"提升了某事件的置信度"等机构语言
2. 禁止直接荐股：只能输出资产类别（如美债）、宽基/行业ETF或产业龙头特征，严禁推荐单一代码
3. 强制风险对冲：每一个推演必须包含"失效的边界条件"
4. 强制溯源：所有核心事实必须附带有效的URL超链接，格式为 [来源名称](URL)
5. 事实与观点分离：客观新闻与主观推演在排版上必须严格物理隔离

【Output Format】
（Agent需严格按照以下Markdown格式输出，随后转义填入上述JSON的 content 字段中）

🌍 主题一：宏观金融与大类资产
【核心事实】
- [客观总结政策或数据，如：美国1月CPI同比上升3.2%]
- [补充核心细节]
- 信息来源：来源名称A / 来源名称B

【投资推演】
- 资产映射：[如：利空长端美债，推升美元指数]
- 交易级别：[短期冲击 / 中期趋势拐点 / 长期配置逻辑]
- 边界条件与风控：[如：若今晚非农数据爆冷，需警惕债市逼空]

(主题二：中观行业、主题三：地缘政治 严格复用上述结构)
```

## 4. 风险控制与异常处理（Fallback机制）

1. **数据源静默/获取失败**：若未抓取到足以形成推演的重大新闻，Agent需输出"当前时段信息静默，无重大宏观异动"，严禁大模型产生幻觉强行捏造分析与虚假URL
2. **API调用与网络异常**：若飞书推送失败，需在本地日志记录Error，建立重试机制（如3次失败则放弃），严禁引发接口死循环
3. **长文本截断风险**：若单次生成的信息量超过飞书卡片Markdown节点限制，Agent需执行摘要压缩，优先保障"投资推演"与"边界条件"透出
